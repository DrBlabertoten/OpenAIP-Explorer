# Product Management Lifecycle (PMLC)
## Status: Phase 1: Ideation (Active)
## Primary Framework: Jobs to be Done (JTBD)
## Agent Directive: AG, refer to the "AG Execution" block under each phase for your specific operational instructions.

### Phase 1: Ideation – Laying the Foundation
Significance: Identifying the "Gap" to avoid building for non-existent problems.
Frameworks: * JTBD: Focus on the user's ultimate goal.

First Principles: Break problems into basic truths.

🤖 AG Execution (Phase 1):
Research: Use the browser to scrape forums (Reddit/Aviation Discord logs) for user "struggles."

Synthesis: Generate a JTBD Canvas (Functional, Emotional, Social jobs).

Validation: Create a Google Form to validate these "Gaps" with real users.

### Phase 2: Value Proposition
Significance: Defining the "Unique Hook" over existing tools.
Frameworks: * Value Prop Canvas: Mapping Pains/Gains to our features.

The 5 Whys: Drilling down to emotional value.

🤖 AG Execution (Phase 2):
Drafting: Generate a Value Proposition statement using the formula: “For [User], who [Struggle], our [Product] is a [Category] that [Value].”

Comparison: Use the browser to find 3 competitors and create a "Pains vs. Relievers" table.

### Phase 3: Prototype (POC) – Testing Feasibility
Significance: De-risking the technical core (SAAP - Stabilize & Assess).

🤖 AG Execution (Phase 3):
Coding: Build the "Minimal Functional Logic" (e.g., the Python script to pull VATSIM API data) before any UI work.

Assessment: If the API fails, AG must suggest 2 alternative data sources immediately.

### Phase 4: Design (Blueprint) – Structuring
Significance: Information Architecture and User Flow.

🤖 AG Execution (Phase 4):
Diagramming: Generate Mermaid.js code for the "Happy Path" user flow.

Documentation: Create a CONTRIBUTORS.md file using the RACI Matrix logic.

### Phase 5: Build – Turning Vision into Reality
Significance: Vibe Coding + Engineering Discipline.

🤖 AG Execution (Phase 5):
Task Management: Update the GitHub Kanban board or local TODO.md based on sprint progress.

Review: Perform a "Technical Debt" check every 50 lines of code.

### Phase 6: MVP Launch – Testing the Market
Significance: The "Collision" with reality.

🤖 AG Execution (Phase 6):
Scoping: Use MoSCoW to flag non-essential features for deletion if the deadline is at risk.

Deployment: Use the terminal to push to production/staging.

### Phase 7: Growth & Maintenance
Significance: Scaling and fixing bugs.

🤖 AG Execution (Phase 7):
Monitoring: Analyze user feedback logs for "Happiness" and "Task Success" metrics.

Iteration: Suggest the next "Could-Have" feature based on the Triple Constraint.
